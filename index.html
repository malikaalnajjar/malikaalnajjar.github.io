ESP32MQTTPostgreSQLFastAPIProphetWebSocketPower BIReact

Design an end-to-end IoT monitoring system for retail shelves: real-time sensor data, a scalable backend, and business-ready dashboards, built from scratch as a senior capstone.
Approach
3 ESP32 nodes publish to MQTT, ingested into PostgreSQL (8-table schema, 4+ indexes). FastAPI exposes 30 REST endpoints and a WebSocket hub. Power BI displays 5+ live KPIs. Prophet forecasts restocking needs from 6 months of sensor data.

Sadeed / سديد
Data Quality & Validation Platform

3-layer ETL validation across 3–5 GASTAT government datasets: schema checks (Great Expectations), statistical profiling, ML anomaly detection (One-Class SVM, zero labelled examples). AWS ECS Fargate with Redis caching: 60%+ latency reduction. Arabic LLM reports with zero data leaving the government network.

FastAPIRedisGreat ExpectationsOne-Class SVMAWS ECSOllamaBrowser Ext.
Challenge
Validate data quality for Saudi government (GASTAT) datasets with no labelled anomalies and zero data permitted to leave the government network.
Approach
3-layer pipeline: schema checks via Great Expectations, statistical profiling for drift detection, and One-Class SVM for unsupervised anomaly scoring. Redis caches results. Arabic LLM reports generated on-premise via Ollama on AWS ECS Fargate.
Impact
60%+ latency reduction through Redis caching. Qualified as a GASTAT Hackathon finalist. Zero data exfiltration by design, meeting strict government compliance requirements.
